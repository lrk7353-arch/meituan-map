# 大学城美食地图 v2.0 - 设计理念说明

## 📋 版本更新概述

v2.0 版本基于产品理念的深度思考进行了全面重构，从"列表浏览"转变为"地图探探"模式，更符合真实的用餐决策场景。

---

## 🎯 核心设计理念

### 1. 地图是核心，不是附加功能

**问题洞察：**
- 原版本虽然叫"美食地图"，但地图只是辅助功能
- 题目明确要求"地图网页应用"，地图应该是核心载体

**v2.0 解决方案：**
- 餐厅信息以**气泡形式**直接显示在地图上的实际位置
- 用户可以看到餐厅的地理分布，理解空间关系
- 每次选择后地图自动平移到下一家餐厅位置
- 地图标记高亮显示当前推荐餐厅

---

### 2. 用餐决策是感性的，不是理性的

**问题洞察：**
- 原版本的列表对比形式像"学术研究"
- 真实场景中，"想吃什么"是冲动和感觉驱动的
- 除非有严格的预算或热量控制，否则人们不会对比雷达图

**v2.0 解决方案：**
- **探探式交互** - 喜欢/不喜欢的直观选择
- **底部控制栏** - 大大的❌和✅按钮，符合直觉
- **每次只看一家** - 避免选择 paralysis
- **地图平移动画** - 增强空间感和探索感

---

### 3. 场景化推荐，而不是通用推荐

**问题洞察：**
- 不同场景下用户需求完全不同
- 减肥期间 vs 放纵日 vs 囊中羞涩时的选择标准不同

**v2.0 解决方案：**

#### 三种决策模式

| 模式 | 目标用户 | 推荐策略 |
|------|---------|---------|
| 🥗 控制热量 | 减肥/健康意识 | 轻食、沙拉、低卡选项 |
| 💰 控制预算 | 学生党/省钱 | 平价餐厅、性价比高的店 |
| 🎲 随意 | 选择困难/随缘 | 基于位置 + 好评推荐 |

---

### 4. 增加趣味性和决策效率

**问题洞察：**
- 选择困难症用户需要外部推动
- 随机性可以增加趣味性

**v2.0 解决方案：**

#### 幸运转盘功能
- 10 种美食品类：火锅、烧烤、日料、韩餐、西餐、中餐、奶茶、甜品、快餐、轻食
- 旋转动画增加期待感
- "再抽一次"或"立刻推荐"两种选择
- 根据抽取结果智能推荐对应品类的餐厅

---

### 5. 与美团生态无缝衔接

**问题洞察：**
- 这是美团的招聘项目
- 用户最终需要完成"吃"这个行为
- 团购和外卖是美团的两大核心业务

**v2.0 解决方案：**

#### 用餐方式切换
- 🏪 **到店吃** - 显示"查看团购"按钮
- 🛵 **外卖** - 显示"点外卖"按钮

#### 跳转逻辑
- 选中餐厅后，可直接跳转到美团对应页面
- 团购 → 美团团购搜索页
- 外卖 → 美团外卖搜索页

**技术实现：**
```javascript
// 团购
https://i.meituan.com/search?keyword={餐厅名称}

// 外卖
https://waimai.meituan.com/search?keyword={餐厅名称}
```

**未来可扩展：**
- 美团 Deep Link（移动端）
- 美团 API 集成（需要合作）
- 团购券购买、外卖下单

## 🔄 用户流程

### 标准流程
```
1. 打开应用 → 选择模式（热量/预算/随意）
2. 进入地图页 → 看到第一家推荐餐厅的气泡
3. 查看气泡信息 → 决定想吃/不想吃
4. 点击❌ → 地图平移到下一家，气泡更新
5. 点击✅ → 收藏餐厅，移动到下一家
6. 点击"团购/外卖" → 跳转美团完成下单
```

### 幸运转盘流程
```
1. 首页点击"打开幸运转盘"
2. 点击"开始旋转"
3. 等待 3 秒动画 → 显示结果（如"火锅"）
4. 选择"立刻推荐" → 跳转到地图页，筛选出火锅类餐厅
5. 或选择"再抽一次" → 重新旋转
```

## 📊 技术实现亮点

### 1. 状态管理（Pinia）
```javascript
// 核心状态
selectMode: 'health' | 'budget' | 'random'
diningType: 'dine-in' | 'delivery'
luckyDrawResult: string | null
currentRecommendIndex: number

// 计算属性
getFilteredRestaurants: 根据模式和转盘结果筛选
getCurrentRecommendation: 获取当前推荐的餐厅
```

### 2. 地图交互
```javascript
// 地图平移到餐厅位置
map.panTo([lng, lat])
map.setZoom(16)

// 高亮当前餐厅标记
custom-marker.active { transform: scale(1.2) }
```

### 3. 本地存储持久化
```javascript
// 记住用户偏好
localStorage.setItem('foodie_mode', mode)
localStorage.setItem('dining_type', type)
```

## 💡 总结

v2.0 版本的核心理念是：

> **"让选择变得简单有趣，让地图成为决策的核心载体"**

我们不再让用户在列表中对比参数，而是通过地图的 spatial 展示、探探式的直观选择、场景化的智能推荐，帮助用户快速做出"今天吃什么"这个永恒难题的决策。

同时，通过与美团生态的衔接，我们完成了从"发现"到"消费"的完整闭环，这正是美团作为本地生活服务平台的核心价值。
