# 大学城美食地图 - 产品设计方案

## 一、产品定位

面向高校学生的本地化美食发现平台，解决"吃什么"的决策困境，提供便捷的餐厅信息和评价体系。

## 二、核心功能设计

### 2.1 地图浏览
- 在地图上标记餐厅位置
- 点击标记显示餐厅基本信息
- 地图缩放、拖拽交互

### 2.2 分类筛选
- 按菜系：中餐、西餐、日韩料理、火锅烧烤、奶茶甜点等
- 按价格：人均消费档次
- 按距离：距离当前位置的距离
- 按评分：用户评价星级

### 2.3 搜索功能
- 关键词搜索（餐厅名、菜品）
- 地区/商圈搜索

### 2.4 餐厅详情
- 基本信息：名称、地址、营业时间、联系方式
- 菜品图片展示
- 用户评价和评分
- 人均消费
- 推荐菜品

### 2.5 评价系统
- 用户可添加评价和打分
- 上传菜品图片（模拟）
- 点赞/踩功能

### 2.6 个人中心
- 我的收藏
- 我的评价
- 浏览历史

## 三、技术方案

### 3.1 前端技术栈
- **框架**: Vue.js 3 (Composition API)
- **地图**: 高德地图 API (免费、支持国内)
- **UI组件库**: Element Plus
- **状态管理**: Pinia
- **构建工具**: Vite

### 3.2 后端技术栈
- **运行时**: Node.js + Express
- **数据存储**: JSON文件（轻量化，适合演示）
- **API设计**: RESTful风格

## 五、开发计划

### Phase 1: 基础架构搭建
- 前后端项目初始化
- 数据模型设计
- 基础API接口

### Phase 2: 核心功能开发
- 地图集成和餐厅标记
- 餐厅列表展示
- 搜索和筛选功能

### Phase 3: 详情页和评价系统
- 餐厅详情页
- 评价展示和添加

### Phase 4: 优化和美化
- UI/UX优化
- 响应式适配
- 性能优化

## 六、创新点

1. **学生专属标签**: 突出显示适合学生的优惠、套餐
2. **智能推荐**: 根据浏览历史推荐相似餐厅
3. **室友拼单**: 提示附近同学正在拼单的餐厅（模拟功能）
4. **饭点提醒**: 根据上课时间推荐合适的餐厅
